VIRTUAL HEALTH ASSISTANT (MVP DEMO)

HealthPulse AI is a local MVP for an AI-powered virtual health assistant for
real-time patient monitoring and support.

Important: this software is for demo/educational use only. It is not a medical
device and must not be used for diagnosis, treatment decisions, or emergency
care.

Project layout
- frontend/: Modern HTML/JS dashboard and assistant UI (served by the backend)
- backend/: FastAPI + SQLite API
- backend/app/main.py: API entry point and static file server
- backend/app/prediction_engine.py: explainable MVP risk scoring and trend forecast
- backend/app/ml_model.py: PyTorch Transformer sequence classifier loader
- backend/app/chat_service.py: NLP chat using Hugging Face Flan-T5-small with keyword fallbacks
- backend/app/evaluation.py: embedded alert-rule evaluation harness
- backend/scripts/train_health_model.py: reproducible dataset generation and Transformer training
- backend/ml/: saved trained model (.pth) and model evaluation metrics
- data/: sample CSV file for upload testing
- data/synthetic_labeled_sequences.json: labeled training/evaluation dataset generated by the training script

Requirements
- Python 3.12+
- PyTorch and Hugging Face Transformers for ML prediction and NLP chat
- Fallback keyword responses work when model dependencies are unavailable

Recommended setup
1. Activate the backend venv.
   .\backend\.venv\Scripts\Activate.ps1
2. Install dependencies.
   pip install -r requirements.txt
3. Configure environment variables.
   Copy backend\.env.example to backend\.env or create a root .env.

Run the application
1. Open a terminal in the backend folder.
2. Run:
   .\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000
3. Open:
   http://127.0.0.1:8000
   (The backend serves the frontend at the root URL)

Backend endpoints
- GET /api/health
- GET /api/vitals?limit=100
- POST /api/vitals
- POST /api/vitals/upload-csv
- POST /api/vitals/simulate
- GET /api/alerts?limit=50
- GET /api/prediction?limit=24
- GET /api/summary
- GET /api/evaluation
- POST /api/chat
- GET /api/chat/history?limit=50
- DELETE /api/chat/history

CSV format
Recommended headers:
- recorded_at,heart_rate,bp_systolic,bp_diastolic,spo2,body_temp_c

Accepted aliases:
- heart_rate: hr, bpm
- bp_systolic: systolic, sys
- bp_diastolic: diastolic, dia
- spo2: sp_o2, oxygen
- body_temp_c: temperature, temp_c
- recorded_at: timestamp, time

Quick demo flow
1. Start the FastAPI backend.
2. Open http://127.0.0.1:8000 in your browser.
3. Watch simulated vitals update and sync to the backend.
4. Inspect alerts, risk score, and the risk trend.
5. Ask the assistant a status or symptom question in the chat interface.
6. Open /api/summary and /api/evaluation for defense/demo evidence.

Train or refresh the ML model
1. Activate the backend venv.
2. Run:
   python backend\scripts\train_health_model.py
3. Generated outputs:
   - data\synthetic_labeled_sequences.json
   - backend\ml\health_risk_model.pth
   - backend\ml\model_metrics.json

Implementation objective coverage
- Patient interaction: Unified HTML/JS assistant with persistent chat history.
- Real-time monitoring: Dashboard with live vitals simulation and ingestion.
- Health prediction: PyTorch Transformer sequence classifier plus explainable risk score and next-value estimate.
- Evaluation metrics: /api/evaluation reports alert-rule precision/recall/F1 and trained model accuracy/macro-F1.
- Comparison readiness: /api/evaluation and PROJECT_ALIGNMENT.md summarize how the MVP compares with common solution types.

Out of scope
- Production hardening and compliance
- Clinical diagnosis logic
- Native wearable/Bluetooth integrations
- Full thesis writing
